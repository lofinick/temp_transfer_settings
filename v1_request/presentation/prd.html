    Part 1: 架构、边界与用户故事 (Product Architecture & User Stories)
    🔗 SSOT: 01_product/overview.md | 01_product/user_stories.md

    用户任务与 MVP 范围

    用户来到 Transfer Settings，主要为了完成：

    用户任务 对应模块
    查看账户和存储状态Account
    设置录音/文件同步保留策略Sync
    连接 Notion / Google Drive 等导出目的地Export
    设置 AI 自动处理偏好AI Boost (CLI 扩展)
    处理异常：空间满、授权失效、隐私设置确认Account / Export

    MVP In Scope：

    • 四个 Tab 的基础信息架构
    • 存储容量展示与 90% / 100% 状态
    • Retention 设置
    • Notion 授权状态展示与重授权
    • data training opt-in 默认关闭

    Out of Scope for v1：

    • Billing Hub 完整购买流程
    • 多个第三方集成的完整管理
    • 团队/企业权限管理
    • 复杂 AI workflow builder
    • 历史导出日志与失败重试队列可视化

    网页端控制权 (Web Transit Station)

    提供对多端通用的云资源、第三方集成与 AI 工作流的治理。呈现为四大平滑切换的 Tab：Account、Sync、Export、AI Boost。

    1. 存储空间熔断与恢复闭环 (Storage Quota Flow)

    stateDiagram-v2 [*] --> Normal : 账户初始化 Normal --> Warning : 容量到达 90% Warning --> Normal : 用户主动清理文件 Warning --> Blocked : 硬件持续并发写入致 100% Normal --> Blocked : 大文件单次突增致 100% Blocked --> Normal : 前往 Billing Hub 购买扩容 Blocked --> Normal : 用户主动清理文件
    • Warning 触发：当达到 90% 时，网页端前端获取到信号，UI 静默渲染红色警示进度条。此时硬件上传不断联。
    • Blocked 阻断：突破 100% 阈值的瞬间，API 网关强制拒绝一切硬件的 POST 请求，统一返回 HTTP 403，Body 必须包含以下结构以便前端准确判断异常类型： 
      {
  "error_code": "storage_quota_exceeded",
  "message": "Storage quota exceeded",
  "current_usage_bytes": 1000000000,
  "quota_bytes": 1000000000
}
      网页端同步状态标记为 Blocked。

    2. 第三方授权自愈闭环 (Integration Token Flow)

    stateDiagram-v2 [*] --> Disconnected : 初始未授权 Disconnected --> Connected : 用户走完 OAuth 流程 Connected --> Token_Expired : 第三方撤销或安全拦截 (401) Token_Expired --> Connected : 用户点击 Re-auth 重新授权 Token_Expired --> Disconnected : 用户点击彻底解绑
    • 隐式降级：后端被 Notion 等接口阻断抛出 401 时，必须立刻将该用户设置表状态强置为 token_expired。
    • 前端捕获与自愈：接口返回 token_expired，前端屏蔽正常的管理界面，高亮显示橙色的重授权按钮，引导用户完成二次握手。

    3. 数据同步与保存机制

    • 自动保存与防抖：所有设置项采用无刷新自动保存，连续操作触发 >500ms 的防抖机制，避免竞态条件。
    • 状态反馈：保存中必须有 Loading 反馈；保存失败必须将开关 UI 回弹并弹出全局报错 Toast。
    • 轮询策略：针对空间容量等强相关指标，用户停留页面时需开启长轮询或短轮询刷新数据。

    交互边界与响应式规范

    • 初次加载：要求使用灰色渐变骨架屏 (Skeleton) 平滑过渡，禁止白屏闪烁。
    • 异常视觉：Notion 授权过期时，强制渲染为浅橙色背景的 Re-authenticate 专属卡片。
    • Toast 组件：定义了 z-index: 9999 的全局轻量提示，成功停留 2s，失败停留 4s 且伴随 UI 强制回弹。
    • 移动端降级 (Mobile)：< 768px 时，Sidebar 降级为汉堡菜单 + 抽屉，内页 Nav 降级为顶部水平滑动 Tab。
    Part 4: 验收标准 (QA Edge Cases)
    🔗 SSOT: 04_checks/qa_guidelines.md

    1. 前端交互与状态自愈防线

    • 状态不同步与回弹：用户点击开启 Opt-in 的瞬间断网，UI 开关必须在 Catch 到报错后平滑回弹至关闭状态，严禁出现“假开启”。
    • 授权失效的隐式捕获：后台修改 token_expired 状态，前端必须立刻拦截设置更改，渲染橙色的重新授权卡片。

    2. 后端并发与原子阻断防线

    • 并发写满的竞态防御：当空间剩 10MB 时，并行打入两个 8MB 文件，系统必须采用原子化的容量预占/提交机制，确保并发上传时不会突破用户容量上限。具体实现可由工程侧决定，例如数据库事务、行锁、Redis 分布式锁或对象存储回调校验。有且仅有一个请求能进，另一个强阻断并报错 403。
    • 定时任务清理安全：Retention 的 Cron Job 批量执行删除时，不能锁表卡死用户的其他设置读写操作。

    3. 隐私基线

    • 任何新用户的 data_training_opt_in 字段绝对不得在初始插入数据库时被赋值为 true。
