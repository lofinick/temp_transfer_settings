# 核心用户故事 (User Stories)

本章节补充了中转站设置 (Transfer Settings) 的核心业务场景，帮助产研团队理解“用户为什么要做这些操作”，以及在极端情况下软硬件的协同体验。

## 场景 1：存储空间爆满与软硬件协同 (Storage Quota Exceeded)

**背景**：
用户的云端存储空间 (10GB) 是 Web 端和所有绑定的硬件设备共享的。硬件设备在弱网环境下录音，恢复网络后会自动向云端并发上传录音文件。

**User Story**：
> “作为一名重度会议用户，当我正在使用硬件录音时，如果云端空间满了，我希望设备能有明确反馈，并且我在 Web 端能快速知道发生了什么并解决它，以免丢失重要的会议记录。”

**系统协同体验**：
1. **预警阶段 (90%)**：当空间使用达到 9.2GB 时，Web 端的 Account 页面会出现红色的进度条警告。此时硬件设备**不受影响**，继续正常录音和上传。
2. **熔断阶段 (100%)**：用户在 Web 端或通过硬件持续产生大文件，导致空间达到 10GB 上限。
   - **Web 端**：前端轮询拉取到 `quota_exceeded` 状态，页面全局或 Account 面板出现强阻断提示。
   - **硬件端**：网关强制拒绝硬件的上传请求 (HTTP 403)。硬件必须捕获此错误，**停止上传但不能停止本地录音**（将新录音暂存至本地磁盘），并在硬件 UI（如屏幕或指示灯）上提示“云空间已满”。
3. **自愈阶段**：用户在 Web 端点击“Upgrade Storage Plan”完成支付，或删除了旧的历史文件。Web 端轮询到空间释放，红条消失。硬件设备在下一次心跳或重试时发现上传成功，将本地积压的录音静默同步至云端，恢复稳态。

## 场景 2：第三方授权失效后的安全降级 (Token Expiration & Re-auth)

**背景**：
用户授权 Cuneflow 将会议纪要自动导出到他们的个人 Notion 数据库。Notion 的 Token 可能会因为安全策略（如用户在 Notion 端修改了密码、撤销了授权，或 Token 自然过期）而突然失效。

**User Story**：
> “作为一个依赖自动化工作流的用户，当我的 Notion 授权因为安全原因掉线时，我希望系统不要默默吞掉错误导致我丢失数据，而是能明确提醒我去重新绑定。”

**系统协同体验**：
1. **隐式失效**：系统后台在执行导出任务时，遭遇 Notion API 返回 HTTP 401 (Unauthorized)。
2. **状态降级**：后端严禁直接丢弃任务或仅记录日志，必须立刻将数据库中该用户的 `connections.notion.status` 字段降级为 `token_expired`。
3. **用户感知与自愈**：用户下一次登录 Web 端打开“Export Integrations”面板时，界面不会显示正常的配置项，而是渲染一张显眼的**橙色警告卡片 (Re-authenticate)**。用户点击后重新走一遍 OAuth 流程，状态机闭环恢复至 `Connected`。

## 场景 3：隐私策略的隐性边界 (Privacy Opt-in)

**User Story**：
> “作为一名对数据隐私敏感的用户，我希望默认情况下我的任何数据都不会被用于训练 AI 模型，只有在我明确同意后才可以。”

**系统协同体验**：
- 无论用户是从网页端注册，还是买来硬件直接激活，后端在初始化用户设置记录时，`data_training_opt_in` 必须是 `false` (或 `null` 视同 `false`)。
- 当用户在 Web 端打开此开关时，系统如果因为网络波动保存失败，开关必须在 UI 上回弹至关闭状态，严禁让用户产生“我已经开启了但实际上没开启”的错觉。
